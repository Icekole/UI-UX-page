## 前言

本规范用于统一教育后台设计系统中的基础视觉规则与高频通用组件规则，服务于设计、产品与研发三方协作。文档重点沉淀四类内容：

- `基础 token`：统一颜色、字体、图标、间距、圆角与阴影
- `高频表单组件`：统一输入、选择、日期、数值、滑动等录入控件的状态与尺寸
- `导航与操作组件`：统一按钮、面包屑、分页、下拉菜单、标签页等交互骨架
- `B 端通用场景组件`：统一表格、弹窗、提示等后台高频结构

这份文档的目标不是重新设计一套视觉语言，而是基于当前 token、Figma 组件页与成熟的 B 端组件模式，整理出一套可执行、可复用、可落地的标准规范。

---

## 需求背景

当前设计系统已具备基础 token 和部分 Figma 组件页，但组件规范文档仍然偏基础，存在以下问题：

- 规范只覆盖少量组件，无法支撑后台页面快速扩展
- 不同页面对控件的状态、尺寸、颜色和留白理解不一致
- 设计稿与研发实现之间缺少统一口径，容易出现“同名不同样”
- 一些高频 B 端组件尚无清晰边界，例如选择器、日期控件、表格与弹层

因此需要在原有文档基础上进行系统化扩展，将“基础视觉规则 + 高价值通用组件”统一沉淀为完整的设计规范文档。

## 颜色规范

### 1. 设计原则

- 颜色优先来自 token，不手写临时色值
- 功能色用于语义，不用于纯装饰
- 中性色承担内容层级和结构层级
- 同一语义在亮暗模式下保持角色一致
- Gray 系列主要服务背景和边框，正文文字统一优先使用 Font 系列

#### 列表项与下拉选项全局状态底色规范
为保证系统内长列表、下拉菜单、表格行等大面积交互组件的阅读清晰度与反馈一致性，统一遵循以下背景及图标/文字色彩映射逻辑：
- **常规态 (Normal)**：使用 `Gray 1` 或 `Gray 2`，保证极致的阅读清晰度和界面呼吸感。搭配 `Font 1` 或 `Font 2` 文本。
- **悬停态 (Hover)**：使用 `Gray 3`。大面积或高频滑动时保持中性克制，避免视觉疲劳与色彩噪音。搭配 `Font 1` 文本。
- **选中态 (Selected)**：使用 `Brand 2`。引入品牌色以明确标识当前生效项，兼具品牌表现力与高识别度；文本和激活图标须搭配 `Brand 8`，以确保在加深底色上的最佳视觉对比度。

## 字体规范

### 1. 字体栈

```css
font-family: 'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', Arial, sans-serif;
```

### 2. 基础字号

| 变量 | 字号 | 行高 | 建议用途 |
|---|---:|---:|---|
| `font-size-xs` | 12px | 18px | 注释 / 辅助信息 |
| `font-size-sm` | 12px | 20px | 小字号说明 |
| `font-size-base` | 14px | 22px | 默认正文 |
| `font-size-lg` | 16px | 24px | 次级标题 / 强调正文 |
| `font-size-xl` | 20px | 28px | 模块标题 |
| `font-size-2xl` | 24px | 32px | 页面标题 |
| `font-size-3xl` | 30px | 38px | 大标题 |
| `font-size-4xl` | 38px | 46px | 超级标题 |

### 3. 字重

| 变量 | 数值 | 建议用途 |
|---|---:|---|
| `font-weight-regular` | 400 | 正文、描述 |
| `font-weight-medium` | 500 | 按钮文字、标签、表单字段 |
| `font-weight-semibold` | 600 | 标题、强调 |
| `font-weight-bold` | 700 | 强调数字、关键标题 |

### 4. 语义层

#### Title

- 用于页面标题、模块标题、卡片标题
- 同一页面标题层级不超过 3 个等级
- 默认倾向 `semibold` 或 `bold`

#### Body

- 用于正文、说明文案、表格内容
- 默认使用 `regular`
- 需要强调时可提升为 `medium`

#### Label

- 用于按钮、标签、表单字段名、控件标题
- 通常使用 `medium`
- 小尺寸场景可降为 `regular`

### 5. 使用规则

- 正文尽量保持 14px 为基准
- 输入类控件优先使用 14px，紧凑态允许 12px
- 页面级大标题和数据型大数字不宜同时过多出现
- 中文场景以可读性优先，不追求过细字重

## 图标规范

### 1. 组件目标

- 用于表达操作、状态、导航方向和信息提示
- 作为按钮、表单、标签、分页、空状态中的辅助视觉元素
- 提升界面识别效率，但不替代文字语义

### 2. 组件现状

- 当前 Figma 图标页沉淀 `1666` 个图标组件
- 图标基础资源尺寸统一为 `24px × 24px`
- 当前规范只定义图标使用规则，不逐个枚举图标库

### 3. 尺寸规则

| 使用场景 | 推荐尺寸 | 说明 |
|---|---:|---|
| 按钮内图标-大 | 16px | 对应 40px 高按钮 |
| 按钮内图标-默认 | 16px | 对应 32px 高按钮 |
| 按钮内图标-小 | 12px | 对应 24px 高按钮 |
| 表单前后缀图标 | 14px - 16px | 与输入框高度协调 |
| 状态图标 | 16px - 20px | 视信息层级决定 |
| 原始资源尺寸 | 24px | 资源基底，不等于最终展示尺寸 |

### 4. 颜色规则

| 场景 | 颜色规范 |
|---|---|
| 默认操作图标 | `Font 3 / #616161` |
| 重要强调图标 | `Brand 6 / #02A999` |
| 悬停操作图标 | `Brand 7 / #019288` |
| 选中 / 激活图标 | `Brand 6 / #02A999` |
| 危险图标 | `Red 6 / #F53F3F` |
| 成功图标 | `Green 6 / #00B42A` |
| 警告图标 | `Orange 6 / #FF7D00` |
| 禁用图标 | `Font Disable / #BDBDBD` |
| 反白场景图标 | `Font Selected / #FFFFFF` |

### 5. 使用规则

- 图标优先承担辅助识别作用，不单独承担关键语义
- 纯图标按钮必须保留足够点击热区，不按图形本身尺寸计算
- 图标与文字组合时，默认间距使用 4px 或 8px
- 同一区域内线性图标和填充图标不要混用过多
- 危险、成功、警告等语义色只用于有明确状态含义的图标
- 如果图标会单独触发操作，应补充 Tooltip 或相邻文字说明

## 布局规范

### 1. 间距

| 变量 | 数值 | 建议用途 |
|---|---:|---|
| `spacing-xs` | 4px | 图标与文字间距、细微分隔 |
| `spacing-sm` | 8px | 组件内部常规间距 |
| `spacing-md` | 16px | 模块内边距 |
| `spacing-lg` | 24px | 区块间距 |
| `spacing-xl` | 32px | 页面级间距 |
| `spacing-2xl` | 48px | 大区块间距 |
| `spacing-3xl` | 64px | 页面主视觉留白 |

### 2. 圆角

| 变量 | 数值 | 建议用途 |
|---|---:|---|
| `radius-sm` | 4px | 紧凑组件 |
| `radius-md` | 6px | 默认控件 |
| `radius-lg` | 8px | 卡片 / 按钮 |
| `radius-xl` | 12px | 大容器 |
| `radius-full` | 9999px | 胶囊形组件 |

### 3. 布局规则

- 页面外层建议使用 24px 以上留白
- 组件内部间距以 8px、16px 为主
- 同一行内按钮间距优先 8px
- 表单项上下间距建议 16px 到 24px
- 图标与文字间距优先 4px 或 8px
- 圆角不混用，默认组件尽量保持单一圆角级别

## 阴影规范

### 1. 组件目标

- 用于表达层级、浮层和悬浮关系
- 适用于卡片、弹层、下拉菜单、抽屉等

### 2. 阴影层级

| 层级 | Position X | Position Y | Blur | Spread | Color | 建议用途 |
|---|---:|---:|---:|---:|---|---|
| `shadow-sm` | 0 | 1 | 2 | 0 | `rgba(0, 0, 0, 0.05)` | 小卡片、悬浮按钮 |
| `shadow-md` | 0 | 4 | 8 | 0 | `rgba(0, 0, 0, 0.08)` | 常规卡片、下拉层 |
| `shadow-lg` | 0 | 8 | 24 | -2 | `rgba(0, 0, 0, 0.10)` | 弹窗、抽屉 |
| `shadow-xl` | 0 | 12 | 32 | -4 | `rgba(0, 0, 0, 0.12)` | 最高级浮层 |

### 3. 使用规则

- 页面默认不要滥用阴影
- 阴影主要用于“浮起来”的组件，不用于装饰
- 同一产品内阴影层级要保持稳定
- 暗色模式下阴影应更克制，避免发灰发脏
- 最小阴影统一为 `0 1px 2px 0 rgba(0, 0, 0, 0.05)`

| 尺寸 | 高度 | 内边距 | 图标尺寸 | 字号 / 行高 | 间距 |

| 大 | 40px | `12px 8px` | 16px | 16px / 24px | 8px |
| 默认 | 32px | `12px 5px` | 16px | 14px / 22px | 6px |
| 小 | 24px | `12px 2px` | 12px | 12px / 20px | 4px |

### 2. 类型范围

- 主要按钮
- 次要按钮
- 文字按钮
- 危险按钮

### 3. 通用规则

- 所有按钮统一使用 `font-family: PingFang SC`
- 按钮文字统一使用 `Medium / 500`
- 主 / 次 / 文字 / 危险按钮都遵循同一套尺寸规则
- 图标按钮与文字按钮共享相同的圆角和高度基线
- 小尺寸按钮优先用于辅助工具位，不用于主流程提交

### 4. 视觉规范

#### 主要按钮

| 状态 | 背景色 | 边框色 | 文字色 | 图标色 |
|---|---|---|---|---|

| 悬停 | `Brand 7 / #019288` | 无 | `Font Selected / #FFFFFF` | `Font Selected / #FFFFFF` |
| 按下 | `Brand 8 / #017B77` | 无 | `Font Selected / #FFFFFF` | `Font Selected / #FFFFFF` |
| 禁用 | `Gray 3 / #E5E6EB` | 无 | `Font Disable / #BDBDBD` | `Font Disable / #BDBDBD` |

## 富文本编辑器规范

### 1. 组件目标

- 用于长文本、富文本（如加粗、斜体、对齐、列表、图片插入等）的内容排版与录入
- 适用于课程简介、文章编辑、通知公告、作业说明等复杂文本创作场景
- 作为一个独立的复合型组件，从“输入框 (Inputs)”体系中抽离，但外层容器的交互状态映射保持与输入框一致

### 2. 结构拆解

富文本编辑器由以下核心区域构成：
- 外层容器 (Container)
- 工具栏 (Toolbar)：内含格式化按钮、下拉菜单及分割线
- 编辑区 (Content Area)

### 3. 状态定义与交互规范

外围交互状态继承系统输入控件统一体系，工具栏内部状态提供独立的高对比度操作反馈。

#### 外层容器 (Container)

| 状态 | 边框色 | 背景色 | 其它效果 |
|---|---|---|---|
| **默认** | `Gray 4 / #C9CDD4` | `Gray 1 / #FFFFFF` | 无 |
| **悬停** | `Brand 6 / #02A999` | `Gray 1 / #FFFFFF` | 无 |
| **聚焦** | `Brand 6 / #02A999` | `Gray 1 / #FFFFFF` | Brand 体系的光晕阴影 |
| **禁用** | `Gray 3 / #E5E6EB` | `Gray 2 / #F2F3F5` | 编辑区与工具栏不可操作 |

#### 工具栏操作项 (Toolbar Item)

| 状态 | 背景色 | 图标 / 文本色 | 说明 |
|---|---|---|---|
| **默认** | `透明` / 无背景 | `Font 3 / #616161` | 普通静默状态 |
| **悬停** | `Gray 3 / #E5E6EB` | `Font 1 / #242424` | 鼠标经过按钮 |
| **选中** | `Brand 2 / #AEEEDF` | `Brand 8 / #017B77` | 格式已激活（如当前光标处已是加粗） |
| **禁用** | `透明` / 无背景 | `Font Disable / #BDBDBD` | 整体禁用或操作冲突不可用时 |

### 4. 尺寸与间距规则

- **外层圆角**：统一继承体系内的圆角变量（如 `radius-3` 或 `radius-5`，对齐 Inputs 规范）。
- **工具栏内边距**：建议使用 `spacing-6` (12px) 或 `spacing-10` (20px)，保证操作区舒适。
- **工具间距**：相邻普通图标按钮之间的 `gap` 保持紧凑，建议 `spacing-2` (4px)。
- **工具栏组分割线**：使用 `Gray 3 / #E5E6EB` 绘制 1px 竖线分隔不同类别（如字形组、排版组、插入组），左右留距各 `spacing-5` (10px)。
- **工具栏与编辑区分割线**：使用 `Gray 3` 或 `Gray 4` 贯穿横线切分，保持视觉整洁。

### 5. 使用规则

- 工具栏功能应按“业务实际需要”精简，避免堆砌过于复杂的生僻功能。
- 当视窗变窄或在小屏模式下，过多的工具按钮应自动聚合为“更多 (More)”的下拉面板。
- 富文本由于承载内容繁多，务必设定合适的**最小高度**，并在内容溢出时触发编辑区自身的垂直滚动条，而不应无限撑高破坏系统页面布局。

## 左侧导航菜单（左树）规范

### 1. 组件目标

- 用于承载应用全局或模块局部的系统导航结构
- 提供清晰的功能入口，并可支撑多层级的子功能展现
- 采用排他型手风琴交互，保持侧边栏信息密度的平衡

### 2. 组件结构

左侧导航菜单（左树）不再使用单一大组件集承载所有场景，而是拆分为两个职责清晰的组件集：

- **左树-普通项**：用于单行导航节点。
  - **节点状态**：`默认` / `悬停` / `选中`。
  - **布尔与文本属性**：`显示左图标`、`文案`。
- **左树-级联组**：用于 `1 个父项 + 2~5 个平级子项` 的级联结构。
  - **子项数量**：`2项` / `3项` / `4项` / `5项`。
  - **展开状态**：`展开` / `收起`。
  - **当前节点**：`父项` / `子项1` / `子项2` / `子项3` / `子项4` / `子项5`。
  - **节点状态**：`默认` / `悬停` / `选中`。
  - **布尔与文本属性**：`显示左图标`、`父项文案`、`子项1文案` 至 `子项5文案`。

*注：普通项不再暴露“右侧元素”属性。级联组的子项不再暴露图标类属性，统一不显示左侧主图标、右侧操作图标和右侧展开箭头，只保留树形连接线与文本内容。*

### 3. 尺寸规则

| 元素 | 高度 | 内边距 | 字号 / 字重 | 圆角 |
|---|---:|---:|---|---:|
| 父级菜单项 | 40px | `8px 10px` | 14px / Medium | 4px |
| 子级菜单项内容区 | 40px | `8px 10px 8px 0` | 14px / Regular | 4px |
| 子级菜单项连接线区 | 40px | `0` | - | 0px |

*注：级联子项改为“左侧连接线区 + 右侧内容区”的左右布局。连接线区固定承担树形折线，背景色只作用于右侧内容区，避免选中或悬停时背景穿过连接线底部。第一项子节点的连接线必须向上连接父级。*

### 4. 状态定义

| 状态 | 说明 |
|---|---|
| **默认** | 可点击激活或展开的入口 |
| **悬停 (Hover)** | 鼠标经过时的弱反馈 |
| **选中 (Selected)** | 当前路由停留页面，是业务当前唯一路径 |

*注：在 `左树-级联组` 中，当 `当前节点` 为任一子项且 `节点状态=选中` 时，父项应自动表现为“父级高亮”效果，但该效果不再作为独立状态暴露在属性面板中。若 `当前节点` 为子项，则组件必须处于 `展开` 状态。*

*补充约束：当 `当前节点` 为任一子项且 `节点状态=悬停` 时，父项必须保持默认态，父子之间同一时刻只允许一个节点处于悬停反馈。*

### 5. 视觉规范

| 层级与状态 | 背景色 | 文字色 | 图标色 |
|---|---|---|---|
| **父级 / 子级：默认** | 透明 | `Font 1 / #242424` | `Font 3 / #616161` |
| **父级 / 子级：悬停** | `Gray 2 / #F2F3F5` | `Font 1 / #242424` | `Font 2 / #424242` |
| **子级：选中 (Selected)** | `Brand 1 / #E8FFF9` | `Brand 6 / #02A999` (加粗) | `Brand 6 / #02A999` |
| **父级：子项选中联动** | 透明 | `Brand 6 / #02A999` (加粗) | `Brand 6 / #02A999` |

*注：父菜单仅当没有子菜单且被直接选中时，才采用背景高亮（等同于子级选中）。一旦父菜单包含子菜单并且某个子项被选中，父级只做文字与图标高亮，不再以独立“半激活”状态形式暴露。级联子项采用“连接线区 + 内容区”的左右布局，背景色仅作用于右侧内容区，避免从连接线底部穿过。*

### 6. 交互使用规则

- **排他型手风琴展开**：默认情况下，“父级菜单”同一时间仅允许展开一项。若用户点击另外一个合拢的父级菜单，先前的父级菜单会自动合拢。这保证了即使用户切换深层级，导航结构也不会变得过度拉长和杂乱。
- **指示器翻转**：右侧箭头必须有明显的动画变化，例如闭合时“向下”展示，展开时“向上”翻转，直观指示当下菜单的展开状态。
- **溢出与滚动**：若内部导航项过多超出侧边容器高度，应在导航内部显示纵向滚动条，而非导致整个页面滚动。
- **属性收口原则**：单行导航项只暴露单行所需属性，级联组只暴露级联所需属性。避免在同一组件集中同时出现“右侧元素”“子项数量”“当前节点”等跨结构属性，降低设计师的判断成本。
- **级联选中约束**：当 `当前节点` 为子项时，组件必须处于展开态；收起态只允许父项作为当前节点。
