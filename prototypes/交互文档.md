# 个人门户网站交互文档

## 一、页面入口

- 用户通过公网地址或本地 HTML 文件进入个人门户，默认展示首页。
- 顶部导航提供四个主视图入口：首页、作品、文章、工具箱。
- 顶部“联系我”按钮可在任意视图打开联系抽屉。
- 首页 CTA 提供三个入口：查看作品、阅读文章、联系我。

## 二、页面规则

### 1. 首页

- 首页首屏展示“产品设计与 AI 原型实践”作为主标题。
- 副标题展示“从 UI/UX 到产品方案落地”。
- 信息标签展示产品设计、产品管理、UI/UX 转产品、AI 工作流、武汉 / 远程协作。
- 首页文案说明能力方向：从用户体验、业务目标到落地协作，转化为产品方案、交互原型和可执行文档。
- 能力摘要展示 UI/UX、PM、AI 三个方向。

### 2. 作品

- 作品视图默认展示全部作品。
- 筛选项包括：全部、产品项目、设计作品、方法论。
- 筛选项互斥激活，同一时间只允许一个筛选项高亮。
- 作品卡片展示类型、标题、摘要、目标或结果说明。
- 作品内容必须按公开展示口径脱敏，不展示真实公司名、内部截图、真实业务数据和不可公开流程。

### 3. 文章

- 文章视图展示文章列表。
- 文章卡片展示标题、摘要、主题标签和阅读时长。
- 当前文章详情通过抽屉展示，不跳转独立文章详情页。
- 文章卡片支持鼠标点击和键盘 Enter 打开详情。

### 4. 工具箱

- 工具箱默认展示设计分类。
- 分类包括：设计、产品、AI、研发。
- 分类互斥激活，同一时间只允许一个分类高亮。
- 资源卡片展示工具名称、用途说明和打开提示。
- 点击资源卡片后在新窗口打开外部链接，当前门户页面保持原状态。

## 三、按钮跳转

| 入口 | 点击结果 |
| --- | --- |
| 顶部“首页” | 切换到首页视图，首页导航高亮 |
| 顶部“作品” | 切换到作品视图，作品导航高亮 |
| 顶部“文章” | 切换到文章视图，文章导航高亮 |
| 顶部“工具箱” | 切换到工具箱视图，工具箱导航高亮 |
| 顶部“联系我” | 打开联系抽屉 |
| 首页“查看作品” | 切换到作品视图 |
| 首页“阅读文章” | 切换到文章视图 |
| 首页“联系我” | 打开联系抽屉 |
| 作品筛选项 | 切换作品分类并刷新作品列表 |
| 作品卡片 | 打开作品详情抽屉 |
| 文章卡片 | 打开文章详情抽屉 |
| 工具箱分类 | 切换工具箱资源分类并刷新资源列表 |
| 资源卡片 | 新窗口打开外部链接 |

## 四、弹窗交互

### 1. 详情抽屉打开

- 点击作品卡片后，抽屉展示作品标题、脱敏详情和目标说明。
- 点击文章卡片后，抽屉展示文章标题、文章简介、主题标签和阅读时长。
- 点击联系入口后，抽屉展示联系说明、联系邮箱和外部平台入口。
- 抽屉打开后，焦点自动移动到关闭按钮。

### 2. 详情抽屉关闭

- 用户点击关闭按钮后关闭抽屉。
- 用户点击遮罩区域后关闭抽屉。
- 用户按 Escape 键后关闭抽屉。
- 抽屉关闭后，焦点回到打开抽屉前的元素。

## 五、状态规则

- 同一时间只允许一个主视图激活。
- 同一时间只允许一个抽屉打开。
- 顶部导航高亮必须与当前主视图保持一致。
- 作品筛选状态在打开和关闭详情抽屉后保持不变。
- 工具箱分类状态在打开外链后保持不变。
- 移动端下作品、文章、工具箱列表切换为单列展示，城市竖排标签隐藏。
- 用户开启减少动效时，页面动画和过渡时间降级到最短。

## 六、空状态和异常规则

- 作品为空时展示“作品正在整理中”。
- 当前作品分类无内容时展示“该分类暂无内容”。
- 文章为空时展示“内容即将上线”。
- 工具箱分类无资源时展示“该分类暂无资源”。
- 外链为空时不展示可点击态，并提示“链接暂未配置”。
- 受控资料不允许直接公开下载，必须通过后续沟通、访问码页面、私密链接或 PDF 附件按需提供。
