# 个人门户网站功能实现文档

## 一、实现范围

- 实现单页个人门户原型，覆盖首页、作品、文章、工具四个主视图。
- 首页视图固定为一屏展示，切换回首页时自动回到顶部且不支持首页内滚动。
- 实现首页公开展示文案，包括个人定位、信息标签、价值陈述、CTA、立体化标题和作品集封面预览。
- 实现作品列表、作品分类筛选和作品详情二级页面。
- 实现基于 2023 UI/UX 作品集的图文案例展示，包括作品封面、本地图片资产、标签、关键指标和结构化详情。
- 实现基于线上文章快照和 Notion CMS 已发布索引的晨读列表和文章详情二级页面，文章栏目不展示说明文案，晨读封面使用 `assets/reading/` 本地图片资产，列表按月份分组并展示月份标题与文章数量，卡片采用两列左图右文布局，左侧封面按 16:9 区域铺满展示，右侧仅展示列表标题和日期，详情页支持放大的列表摘要标题、标题下方日期、标签元信息，并在正文区展示深度文章和资讯动态正文。
- 实现工具全部汇总、分类资源列表和外链打开。
- 实现联系抽屉，用于承接面试、产品设计、AI 原型和合作机会的联系说明。
- 实现桌面端和移动端基础响应式布局。
- 实现基础可访问性能力，包括焦点样式、Enter / Space 进入文章详情、详情页返回后焦点恢复、Escape 关闭联系抽屉和焦点恢复。

## 三、数据流

### 1. 初始化数据流

- 页面加载后读取内置的 `caseAssets`、`readingAssets`、`cases`、`resources` 和 `contactDrawerContent`。
- 页面加载后通过 `loadPublishedReadings()` 先渲染内置线上文章快照，再请求 `content/readings/index.json` 或 `../content/readings/index.json`，该索引由 Notion CMS 的已发布视图同步生成。
- 调用 `renderCases()` 渲染全部作品。
- 调用 `renderBlog()` 渲染已发布晨读封面列表。
- 调用 `renderResources()` 渲染默认全部工具资源。
- 首页视图默认带有 `active` 状态，因此首屏直接展示首页内容。

### 2. 视图切换数据流

- 用户点击带有 `data-view` 的按钮后，读取按钮上的目标视图名称。
- `showView(name)` 遍历所有主视图节点，根据 `data-panel` 与目标视图是否一致切换 `active` 状态。
- `showView(name)` 同步遍历顶部导航按钮，根据 `data-view` 与目标视图是否一致切换高亮状态。
- 当目标视图为 `portfolio-detail` 时，顶部导航仍将“作品”保持为高亮状态。
- 当目标视图为 `article-detail` 时，顶部导航仍将“文章”保持为高亮状态。

### 3. 作品数据流

- 作品数据保存在 `cases` 数组中。
- 作品图片路径保存在 `caseAssets` 对象中，图片文件位于 `assets/portfolio/` 目录。
- 用户点击筛选项后，读取筛选按钮的 `data-filter`。
- `renderCases(type)` 根据筛选类型过滤 `cases`。
- 过滤后的作品数据被渲染为卡片按钮，每张卡片包含封面图、类型、标题、摘要、标签和结果说明。
- 用户点击作品卡片后，将当前作品数据传入 `openCaseDetail(item)`。
- `openCaseDetail(item)` 记录当前焦点到 `lastPortfolioFocused`，并将作品类型、标题、详情、结果和图文内容写入作品详情页。
- `renderCaseDetail(item)` 将当前作品的详情图、图片说明、关键指标和结构化章节写入 `caseDetailContent`。
- `showView("portfolio-detail")` 切换到作品详情二级页面，同时保持顶部“作品”导航高亮。
- 用户点击“返回作品”后触发 `backToPortfolio()`，返回作品列表并恢复进入详情前的作品卡片焦点。
- 作品详情页不改变当前筛选状态。

### 4. 文章数据流

- 文章列表数据进入页面时先来自 `blogPosts` 内置线上文章快照，线上索引读取成功后再替换为 `content/readings/index.json` 中的已发布文章。
- `content/readings/index.json` 只包含 Notion CMS 中状态为“已发布”的文章；草稿和已下架内容不进入线上列表。
- `normalizeReading()` 根据文章日期匹配 `readingAssets` 中的工作日封面图。
- `renderBlog()` 将最近 5 篇已发布晨读文章按月份分组，渲染为两列左图右文的晨读封面卡片。
- 用户点击晨读封面卡片，或在卡片聚焦时按 Enter / Space 后，将当前文章数据传入 `openArticleDetail(post)`。
- `openArticleDetail(post)` 先通过 `fetchArticleDetail(post)` 按 `path` 读取当天详情 JSON。
- 若详情 JSON 中 `status` 不是“已发布”，`openArticleDetail(post)` 不展示详情，并重新加载已发布列表。
- `openArticleDetail(post)` 记录当前焦点到 `lastBlogFocused`，并将文章主题标签、列表摘要标题、日期和正文内容写入文章详情页。
- `renderArticleDetail(post)` 将当前文章的主题标签、阅读时长和正文内容写入 `articleDetailContent`。
- 当文章提供 `contentMarkdown` 时，`extractDetailMarkdown()` 先截取“深度文章 / 报告分析”之后的正文，再由 `markdownToHtml()` 转换为标题、段落、列表和链接。
- 当文章未提供 `contentMarkdown` 时，详情页展示“内容正在同步中”，不复用其他日期的文章正文。
- `showView("article-detail")` 切换到文章详情二级页面，同时保持顶部“文章”导航高亮。
- 用户点击“返回文章”后触发 `backToBlog()`，返回文章列表并恢复进入详情前的晨读封面卡片焦点。

### 5. 工具数据流

- 工具数据保存在 `resources` 对象中，并按分类分组。
- 用户点击工具分类按钮后，读取按钮的 `data-resource`。
- `renderResources(type)` 根据分类读取对应资源数组；当 `type` 为 `all` 时汇总所有工具。
- 每个资源被渲染为外链卡片，外链使用新窗口打开。
- 外链打开不回写页面状态。

### 6. 联系抽屉数据流

- `openDrawer()` 记录当前焦点元素到 `lastFocused`。
- `openDrawer()` 将联系标题、正文和补充内容写入抽屉节点。
- `openDrawer()` 为抽屉添加 `open` 状态，并将 `aria-hidden` 设置为 `false`。
- `closeDrawer()` 移除抽屉 `open` 状态，并将 `aria-hidden` 设置为 `true`。
- `closeDrawer()` 在存在 `lastFocused` 时恢复焦点。
