# 个人门户网站功能实现文档

## 一、实现范围

- 实现单页个人门户原型，覆盖首页、作品、文章、工具箱四个主视图。
- 实现首页公开展示文案，包括个人定位、信息标签、价值陈述、CTA 和能力摘要。
- 实现作品列表、作品分类筛选和作品详情抽屉。
- 实现文章列表和文章详情抽屉。
- 实现工具箱分类资源列表和外链打开。
- 实现联系抽屉，用于承接面试、产品设计、AI 原型和合作机会的联系说明。
- 实现桌面端和移动端基础响应式布局。
- 实现基础可访问性能力，包括焦点样式、Enter 打开文章详情、Escape 关闭抽屉和焦点恢复。

## 二、核心状态

| 状态/数据 | 用途 |
| --- | --- |
| `panels` | 保存所有主视图节点，用于控制首页、作品、文章、工具箱的显示状态 |
| `navButtons` | 保存顶部导航按钮，用于同步当前视图高亮 |
| `cases` | 保存作品数据，包含类型、标签、标题、摘要、结果说明和详情 |
| `blogPosts` | 保存文章数据，包含标题、摘要、主题标签、阅读时长和详情 |
| `resources` | 保存工具箱资源数据，按设计、产品、AI、研发四类组织 |
| `detailDrawer` | 保存详情抽屉节点，用于统一承载作品详情、文章详情和联系信息 |
| `drawerTitle` | 保存抽屉标题节点 |
| `drawerBody` | 保存抽屉正文节点 |
| `drawerExtra` | 保存抽屉补充内容节点 |
| `lastFocused` | 记录打开抽屉前的焦点元素，用于关闭抽屉后恢复焦点 |
| `contactDrawerContent` | 保存联系抽屉内容，包括说明文案、邮箱和外部平台入口 |

## 三、数据流

### 1. 初始化数据流

- 页面加载后读取内置的 `cases`、`blogPosts`、`resources` 和 `contactDrawerContent`。
- 调用 `renderCases()` 渲染全部作品。
- 调用 `renderBlog()` 渲染文章列表。
- 调用 `renderResources()` 渲染默认设计分类资源。
- 首页视图默认带有 `active` 状态，因此首屏直接展示首页内容。

### 2. 视图切换数据流

- 用户点击带有 `data-view` 的按钮后，读取按钮上的目标视图名称。
- `showView(name)` 遍历所有主视图节点，根据 `data-panel` 与目标视图是否一致切换 `active` 状态。
- `showView(name)` 同步遍历顶部导航按钮，根据 `data-view` 与目标视图是否一致切换高亮状态。

### 3. 作品数据流

- 作品数据保存在 `cases` 数组中。
- 用户点击筛选项后，读取筛选按钮的 `data-filter`。
- `renderCases(type)` 根据筛选类型过滤 `cases`。
- 过滤后的作品数据被渲染为卡片按钮。
- 用户点击作品卡片后，将当前作品标题、详情和结果说明传入 `openDrawer()`。
- 抽屉展示作品详情，不改变当前筛选状态。

### 4. 文章数据流

- 文章数据保存在 `blogPosts` 数组中。
- `renderBlog()` 将每篇文章渲染为文章卡片。
- 用户点击文章卡片或在文章卡片聚焦时按 Enter 后，将文章标题、详情、主题标签和阅读时长传入 `openDrawer()`。
- 抽屉展示文章详情，不改变当前主视图状态。

### 5. 工具箱数据流

- 工具箱数据保存在 `resources` 对象中，并按分类分组。
- 用户点击工具箱分类按钮后，读取按钮的 `data-resource`。
- `renderResources(type)` 根据分类读取对应资源数组。
- 每个资源被渲染为外链卡片，外链使用新窗口打开。
- 外链打开不回写页面状态。

### 6. 抽屉数据流

- `openDrawer()` 记录当前焦点元素到 `lastFocused`。
- `openDrawer()` 将传入的标题、正文和补充内容写入抽屉节点。
- `openDrawer()` 为抽屉添加 `open` 状态，并将 `aria-hidden` 设置为 `false`。
- `closeDrawer()` 移除抽屉 `open` 状态，并将 `aria-hidden` 设置为 `true`。
- `closeDrawer()` 在存在 `lastFocused` 时恢复焦点。
